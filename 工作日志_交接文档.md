## 二、本次工作内容

### 2.1 Bug 修复

#### (1) 图片上传按钮状态卡死问题

**问题：** 上传失败时弹出 `alert()` 对话框，但按钮仍显示"上传中..."，用户无法重新上传。

**根因：** `alert()` 是同步阻塞函数，会暂停 JS 执行。代码中先调用 `alert()` 再恢复按钮，导致恢复操作被阻塞。

**修复：** 在 `alert()` 之前恢复按钮状态。

**文件：** `heart-qa.html` 第 858-865 行

```javascript
// 修复前
} else {
  alert('上传失败：' + (data.error || '未知错误'));
  if (btn) { btn.disabled = false; btn.textContent = '上传照片'; }
}

// 修复后
} else {
  if (btn) { btn.disabled = false; btn.textContent = '💌 发给Ta'; }
  alert('发送失败：' + (data.error || '未知错误'));
}
```

同样修复了 `.catch()` 中的网络错误处理。

#### (2) CORS 跨域问题

**问题：** 前端页面在 `localhost:8080`（Python HTTP服务器），API请求发送到 `localhost:3000`（Node服务器），浏览器因CORS策略阻止了跨域请求，导致轮询和上传都失败。

**修复：** 在 `server.js` 中添加 CORS 中间件。

**文件：** `server.js` 第 9-19 行

```javascript
// CORS middleware
app.use((req, res, next) => {
  res.header('Access-Control-Allow-Origin', '*');
  res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
  res.header('Access-Control-Allow-Headers', 'Origin, X-Requested-With, Content-Type, Accept, Authorization');
  if (req.method === 'OPTIONS') {
    res.sendStatus(200);
  } else {
    next();
  }
});
```

#### (3) 文件 input 未隐藏问题

**问题：** CSS选择器 `.photo-upload-area input[type="file"]` 不匹配实际DOM结构（input在upload-area外部），导致原生文件选择控件可见。

**修复：** 使用独立ID选择器 `#momentFile { display: none; }`。

### 2.4 状态跟踪机制

**问题：** 轮询每3秒触发一次，`renderPhotoExchange()` 完全替换 `innerHTML`，导致：
- 用户正在输入备注 → 轮询 → 备注丢失
- 用户已选择文件 → 轮询 → 预览消失

**解决方案：** 类似默契挑战的 `_challengeRenderState` 机制

```javascript
var _momentRenderState = '';     // 'upload' | 'wait' | 'reveal'
var _momentTaJustArrived = false; // Ta刚上传的标记

function renderPhotoExchange() {
  // 确定目标状态
  var targetState = '';
  if (bothUploaded && !_momentTaJustArrived) targetState = 'reveal';
  else if (myPhoto.uploaded) targetState = 'wait';
  else targetState = 'upload';

  // 如果状态未变，跳过重渲染，保留用户输入
  if (targetState === _momentRenderState) return;

  // 状态变化，执行完整渲染
  _momentRenderState = targetState;
  // ... 构建HTML ...
}
```

**状态转换规则：**
- `upload` → `wait`：用户上传成功
- `wait` → `wait`：Ta上传了，但显示通知卡片（不自动跳reveal）
- 通知卡片点击 → `reveal`：用户主动揭晓
- `reveal` → `upload`：点击"再分享此刻"

**具体改动**：

token 格式：8位随机字符串（大小写字母 + 数字，排除易混淆字符）

```javascript
function generateToken() {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZabcdefghjkmnpqrstuvwxyz23456789';
  let token = '';
  for (let i = 0; i < 8; i++) {
    token += chars.charAt(Math.floor(Math.random() * chars.length));
  }
  return token;
}

function validateToken(req, roomData) {
  const clientToken = req.body && req.body.token ? req.body.token : (req.query.token || '');
  if (!roomData.token) return true;                          // 历史数据 → 放行
  const isReadOnly = req.method === 'GET';
  if (!clientToken && isReadOnly) return true;               // 有token的room，无token的GET请求 → 放行
  return roomData.token === clientToken;                     // token匹配 → 放行
}
```

**验证方式**：

1. `curl -X POST http://localhost:3000/api/room/create` 创建房间 → 响应含 `token` 字段
2. `curl http://localhost:3000/api/room/{roomId}` 无 token → ✅ 成功（向后兼容）
3. `curl http://localhost:3000/api/room/{roomId}?token=WRONG` 错误 token → ❌ 403 拒绝
4. `curl http://localhost:3000/api/room/{roomId}?token={correct}` 正确 token → ✅ 成功
5. `curl -X POST .../glimmer` 无 token（POST）→ ❌ 403 拒绝（写操作必须有token）
6. `curl -X POST .../glimmer -d '{"token":"correct"}'` 正确 token → ✅ 成功

**具体改动**：

```javascript
// 修改前
var API_BASE = 'http://localhost:3000';

// 修改后
var API_BASE = (location.hostname === 'localhost' || location.hostname === '127.0.0.1') ? 'http://localhost:3000' : 'http://' + location.hostname + ':3000';
```

**逻辑说明**：
- 本地开发（localhost / 127.0.0.1）→ `http://localhost:3000`
- 局域网访问（手机连电脑 IP，如 192.168.x.x）→ `http://192.168.x.x:3000`（自动取当前页面 hostname）
