## 1. Demo 简介

**是什么：** 慢慢说是一个为两个人打造的私密互动 H5 小空间。手机浏览器打开即用，无需安装 App。两个人各持一把"钥匙"（房间码），进入同一个专属空间，通过每日引导式互动，慢慢靠近彼此。

**面向谁：** 处于亲密关系中的两个人——情侣、夫妻、异地伴侣、密友。尤其是那些因为日常忙碌而沟通变少，又不想让感情变淡的人。

**核心功能：**

| 功能 | 说明 |
|------|------|
| 🌤 微光状态 | 各自记录今天的一个心情瞬间，对方可见，不需要回应。低压力的存在感。 |
| 💭 每日一问 | 每天一个问题，引导双方表达。55 题库分 5 类（情绪/回忆/未来/日常/深度）轮换，避免重复。 |
| 🎯 默契挑战 | 互相猜对方的答案，揭晓后看看你们有多默契。 |
| 💬 真心话 | 一个安全的地方，说出平时不好意思说的话。 |
| 💌 此刻予你 | 每天一个拍照主题，互相分享一张"此刻"的照片，解锁对方镜头里的当下。 |
| 🤲 轻声说 | 情侣悄悄话，深夜的一句话、和解的话、想说但说不出口的话。 |
| 📅 月度回顾 | 月底自动聚合本月互动数据，生成文字总结，看见你们这个月一起走过的路。 |

## 2. Demo 创作思路

**灵感来源：**

这个想法来自一个很真实的观察：很多情侣并不是不爱了，而是"没时间好好说话"。白天各自忙碌，晚上累到只想刷手机，明明躺在床上却像隔着银河。日常沟通变成了"吃了没""早点睡"这种功能性对话，感情慢慢就淡了。

市面上不缺情侣 App，但大多做得很重——要注册、要加好友、要开通知、要每天打卡，反而成了新的压力来源。我想做一个**轻**的东西：不安装、不打扰、不强制，只是给两个人留一个专属的小空间，想起来了就推开门坐一会儿。

**想解决的问题：**

- **沟通变浅**：日常对话只剩功能性的"吃了没"，缺少有温度的表达
- **不知说什么**：想聊点深入的，但不知道从何开口
- **没有仪式感**：感情需要日常的小确认，但容易被忽略
- **异地更难**：见不到面时，需要一个能感受到"对方在"的方式

**为什么做这个方向：**

我做了几个判断：
1. **轻量优于功能堆砌**——H5 即开即用，比 App 门槛低 10 倍
2. **引导式优于自由式**——给一个问题，比"随便聊"更容易开口
3. **异步优于实时**——不要求同时在线，各自有空就记一笔，对方自然看见
4. **私密优于社交**——只有你们两个人，没有点赞、没有围观、没有压力

一个关键取舍：第一版我砍掉了"AI 智能对话"，只做了"AI 预留接口"。因为情侣互动的核心是**两个人之间的真实表达**，AI 介入太多反而会破坏这种私密感。AI 的价值应该是在背后做引导（选题、总结），而不是站在中间当第三者。所以我在 `server.js` 里预留了 `AI_CONFIG` 和 `generateContent()` 函数，当前用本地模板池，未来可以无缝接入云端 AI 做更懂你们的引导。

## 3. Demo 体验方式

本作品提供两种体验方式：

### 方式一：HTML zip（主交付，离线可用）

**下载：** [manmanshuo-demo.zip](上传后替换为社区附件链接)（1.1 MB）

**体验方法：**
1. 下载并解压 zip 文件
2. 双击打开 `index.html`（用手机或电脑浏览器均可）
3. 点击首页的「✨ 立即体验 Demo」按钮
4. 进入演示页，可点击「🔄 切换为Ta」按钮切换视角，体验双向互动

> zip 包含完整的纯前端演示页（demo.html），不依赖后端，双击即用。

### 方式二：在线体验（需后端运行，完整功能）

<!-- 如果部署了在线版本，填这里；否则删除本段 -->
**体验链接：** [在线体验](在线链接占位)

完整功能体验（含房间创建、实时双向同步、照片上传）需后端运行：
1. 访问首页，点击「开启我的空间」创建房间
2. 复制房间码和钥匙，发给 Ta
3. Ta 点击「邀请 Ta 进入」输入房间码加入
4. 两人各自在手机浏览器打开，开始互动

## 4. TRAE 实践过程

本项目全程使用 **TRAE IDE** 开发，AI 辅助完成了从架构设计到功能实现到调试修复的全流程。

### 关键步骤 1：项目架构与后端搭建

用 TRAE IDE 搭建了 Node.js + Express 后端，设计了房间数据结构、双 Token 鉴权机制（房间 Token + 用户认证 Token）、JSON 文件存储方案。AI 帮助生成了 CORS 中间件、Multer 图片上传配置、数据持久化逻辑。

## 5. 技术栈

| 层 | 技术 |
|----|------|
| 前端 | 纯 HTML + CSS + JavaScript（无框架），深色主题 CSS 变量体系 |
| 后端 | Node.js + Express，端口 3000 |
| 存储 | JSON 文件存储（`data/` 目录） |
| 图片上传 | Multer，最大 5MB |
| 鉴权 | 双 Token 机制（房间 Token + 用户认证 Token） |
| 密码安全 | SHA-256 + Salt 哈希 |
| AI | 预留接口，当前本地模板池，未来可接云端 |

## 6. 开发心得

用 TRAE IDE 开发这个项目，最大的感受是**"想到就能做到"**。我没有写一行代码，全部通过自然语言描述需求，AI 帮我把想法变成了可运行的产品。几个印象深刻的地方：

1. **AI 会主动提建议**：比如我说"做个情侣互动 H5"，AI 主动提出了门扉动画、深色主题、双 Token 机制等设计，比我预想的更完整。
2. **边做边 commit**：有一次环境重置导致代码丢失，AI 立刻提醒我"做完一块就 git commit"，之后再也没有丢过。
3. **AI 能处理复杂逻辑**：55 题分类轮换、月度数据聚合、双 Token 鉴权这些复杂逻辑，AI 一次就写对了。
4. **演示思维**：做 Demo 时 AI 主动提出"视角切换"解决单人演示问题，这是我没想到的。

这个项目让我相信，**好的工具不是替你思考，而是帮你说出心里已经有但说不出来的东西**。慢慢说这个产品本身也是这个理念——不是替你表达，而是引导你表达。
